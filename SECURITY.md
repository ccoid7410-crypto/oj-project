# Security Policy

## Supported Versions

이 프로젝트는 별도의 버전 릴리스 없이 `main` 브랜치의 최신 커밋을 운영 서버에 직접 배포합니다.

따라서 보안 업데이트와 취약점 수정은 **항상 `main` 브랜치의 최신 상태만을 기준으로 제공**됩니다. 과거 커밋, 별도 브랜치 및 포크된 버전은 지원하지 않습니다.

| Version | Supported |
| --- | --- |
| `main` 최신 커밋 | :white_check_mark: |
| 과거 커밋, 기타 브랜치 및 포크 | :x: |

## Reporting a Vulnerability

보안 취약점을 발견한 경우, **공개 GitHub 이슈에 등록하지 마세요.**

공개 전 악용될 가능성이 있는 취약점은 충북과학고 재학생 또는 동아리 관계자를 통해 프로젝트 관리자나 개발자에게 비공개로 전달해 주세요.

보고 시 가능한 범위에서 다음 내용을 포함해 주세요.

- 취약점에 대한 설명
- 취약점 재현 절차
- 영향을 받는 기능 또는 데이터
- 예상되는 악용 가능성과 영향 범위
- 관련 로그, 오류 메시지 또는 스크린샷
- 취약점을 발견한 환경과 시점

민감한 개인정보, 인증 정보, 비밀번호, 세션 쿠키, API 키 등은 보고 내용에 포함하지 마세요.

## Response Process

접수된 취약점은 다음 절차에 따라 검토합니다.

- **접수 확인:** 신고 후 3일 이내
- **재현 및 조치 여부 판단:** 신고 후 7일 이내
- **취약점 수정:** 심각도와 영향 범위에 따라 가능한 한 신속하게 진행
- **결과 안내:** 수정 커밋 배포 후 보고자에게 결과 전달

취약점이 아니거나 재현할 수 없는 경우에는 그 사유를 함께 안내합니다.

보고자의 신원과 신고 내용은 요청이 있는 경우 비공개로 유지합니다. 수정이 완료되기 전에는 취약점의 세부 내용이 외부에 공개되지 않도록 협조해 주세요.

## Scope and Limitations

이 프로젝트는 동아리 내부 사용을 목적으로 운영되는 온라인 저지 서버입니다. 따라서 대규모 공개 서비스 수준의 가용성이나 보안 대응 SLA를 보장하지는 않습니다.

다만 접수된 보안 보고는 성실하게 검토하며, 사용자 계정, 제출 코드, 테스트케이스, 관리자 권한 또는 서버 실행 환경에 영향을 미치는 취약점을 우선적으로 처리합니다.
